# 4
commitするときは、一作業ずつ１コミットが原則。

コミットすると、デフォルトでvimが開かれる。-mをすると、vimが開かないのでわからなかったが……。

わかりやすいコミットメッセージの書き方。
１行目：変更内容の要約
２行目：空行
３行目：変更の理由

# ５
削除を反映するには、「git rm ファイル名」。
ブランチは複数開発で使われるもの。
git statusでは、
①ブランチの情報
②origin/master（リモートリポ）との違い
③最後のコミット後からの変更
が分かる。

git statusとgit logの使い分け。
git logで分かること。
①commit:ハッシュ値
※ハッシュ値を使うと、そのコミットの状態まで戻ったりできる。
②Author:
③Date:
④コミットメッセージ

上下移動はj, k。

便利なオプション
--oneline：メッセージのみを一列で表示
-p：変更内容も表示
-p ファイル名：特定のファイルについて、変更内容含めた表示
-n 数字：表示するログ数を指定

# ８
git diff
ファイルの変更差分を表示する。ローカルとステージング領域の差分を表示する。
